## 简介

## 核心组件
- **Song**：音频曲目的元数据载体，包含标题、艺术家、专辑、文件路径、时长、音轨号、专辑封面路径与收藏状态，支持响应式属性变更通知。
- **Playlist**：播放列表，包含名称与歌曲集合。
- **PlayState**：播放状态枚举（停止、播放中、暂停）。
- **PlaybackMode**：播放模式枚举（普通、随机、单曲循环）。
- **PlayerStatus**：播放器状态封装，聚合位置、时长、音量、静音、播放状态与当前曲目，采用不可变设计。
- **AppSettings**：应用程序设置模型，包含扫描文件夹、子文件夹包含、音量、静音状态、最后扫描时间、主题和播放模式等设置。

## 详细组件分析

### AppSettings 模型：应用程序设置管理
- **字段与含义**
  - ScanFolders：扫描文件夹列表，默认为空列表
  - IncludeSubfolders：是否包含子文件夹，默认为 true
  - Volume：音量等级（0-100），默认为 80
  - IsMuted：是否静音，默认为 false
  - LastScanTime：最后扫描时间，默认为空
  - Theme：主题设置，默认为 "Dark"
  - PlaybackMode：播放模式，默认为 "Normal"
- **业务规则**
  - 设置值应在合理范围内（如音量 0-100）
  - 主题和播放模式应为预定义的有效值
  - 文件夹路径应存在且可访问
- **持久化机制**
  - 通过 ConfigurationService 使用 JSON 序列化存储
  - 默认配置文件位于 ApplicationData 目录下的 LocalMusicPlayer 文件夹
  - 支持异步加载和保存操作
- **使用示例与最佳实践**
  - 在应用启动时加载设置，确保用户体验一致性
  - 修改设置后及时保存，避免丢失用户偏好

### Song 模型：增强的音频文件元数据结构
- **字段与含义**
  - Title：曲目标题，支持响应式属性变更通知
  - Artist：第一演奏者（艺人），支持响应式属性变更通知
  - Album：专辑名，支持响应式属性变更通知
  - FilePath：本地文件绝对路径，支持响应式属性变更通知
  - Duration：音频总时长（TimeSpan），支持响应式属性变更通知
  - TrackNumber：音轨编号，支持响应式属性变更通知
  - AlbumArtPath：专辑封面路径，支持响应式属性变更通知
  - HasAlbumArt：是否具有专辑封面的只读属性
  - IsFavorite：是否为收藏歌曲，支持响应式属性变更通知
- **业务规则**
  - FilePath 必须有效且指向可访问的音频文件
  - Title 不应为空；当元数据缺失时需进行回退处理
  - Duration 应由底层媒体库正确解析
  - HasAlbumArt 属性应随 AlbumArtPath 变化而自动更新
- **增强特性**
  - 继承自 ReactiveObject，支持属性变更通知
  - 使用 RaiseAndSetIfChanged 方法确保属性变更的原子性
  - AlbumArtPath 属性变更时自动触发 HasAlbumArt 属性更新
- **数据来源与生成**
  - 通过 FileScannerService 使用 TagLib 解析文件元数据生成 Song
  - 支持扩展名：.mp3、.flac、.wav、.aac、.ogg、.m4a
- **验证与容错**
  - 若解析失败，构造最小可用 Song（仅设置 Title 与 FilePath），避免中断扫描流程
- **使用示例与最佳实践**
  - 在 UI 中显示时，优先使用 Title/Artist/Album，若为空则显示文件名
  - 播放前校验 FilePath 存在性与可读性
  - 收藏功能通过 IsFavorite 属性管理

### Playlist 模型：播放列表与歌曲集合管理
- **字段与含义**
  - Name：播放列表名称，默认为空字符串
  - Songs：歌曲集合（List<Song>），支持动态修改
- **业务规则**
  - 支持添加/移除歌曲；索引越界需进行边界检查
  - 切换当前播放列表时重置当前索引
  - 当 Songs 为空时，Next/Previous 行为需返回失败
- **播放模式下的导航**
  - Normal：顺序播放，到达末尾返回失败
  - Shuffle：随机选择下一曲
  - Loop：循环播放，到达末尾回到开头或反之
- **使用示例与最佳实践**
  - 新建播放列表后立即 SetCurrentPlaylist，确保后续播放逻辑一致
  - UI 中根据 CurrentIndex 高亮当前播放项

### PlaybackMode 枚举：播放模式选项与行为差异
- **定义**
  - Normal：普通顺序播放
  - Shuffle：随机播放
  - Loop：单曲循环（当前曲目）

  - UI 中将"重复"按钮与 Loop 模式关联，"随机"按钮与 Shuffle 模式关联；Normal 作为默认态

### PlayerStatus 模型：播放器状态封装与查询方法
- **字段与含义**

## 依赖分析
- **FileScannerService** 依赖 TagLib 解析元数据，生成 Song 并写入 MusicLibraryService 的集合
- **MusicLibraryService** 提供全局与筛选后的 Song 集合，供视图模型搜索与展示
- **MusicPlayerService** 通过 LibVLC 播放媒体，发布状态事件，供视图模型与 PlaylistService 同步
- **PlaylistService** 维护当前播放列表、索引与播放模式，决定 Next/Previous 行为
- **ConfigurationService** 管理 AppSettings 的持久化，为 SettingsViewModel 提供设置管理
- **MainWindowViewModel** 将上述服务整合，驱动 UI 交互与状态绑定
- **SettingsViewModel** 处理设置界面的交互，通过 ConfigurationService 更新应用设置

## 性能考虑
- **扫描性能**
  - 使用异步扫描与进度报告，避免 UI 卡顿
  - 支持取消令牌，允许用户中断长时间扫描
- **播放性能**
  - 音量与静音切换避免频繁调用底层接口，减少开销
  - 通过定时轮询（如每 500ms）更新 UI，兼顾实时性与性能
- **内存与集合**
  - 使用 ObservableCollection 便于 UI 绑定；注意在大量数据场景下避免频繁重建集合
  - PlayerStatus 使用不可变设计，减少并发访问冲突
- **设置持久化性能**
  - ConfigurationService 使用异步操作避免阻塞 UI 线程
  - JSON 序列化配置文件，支持增量更新
- **元数据解析**
  - 对于损坏或不可解析的文件，快速降级为最小可用 Song，保证扫描继续
- **响应式属性性能**
  - Song 模型的响应式属性变更通知应避免过度频繁的更新

## 故障排查指南
- **扫描无结果**
  - 检查目录是否存在、扩展名是否受支持、是否勾选了子文件夹扫描
- **无法播放**
  - 确认 FilePath 有效且文件存在；检查播放器初始化与 LibVLC 是否成功加载
- **播放结束后不自动下一曲**
  - 确认 PlaybackEnded 事件已订阅；检查 PlaylistService 的 PlayNext 返回值与 CurrentSong 是否为空
- **随机/循环无效**
  - 确认 PlaybackMode 已正确设置；检查索引边界与环绕逻辑
- **UI 不更新**
  - 确认属性绑定与事件订阅（PositionChanged、PlaybackStateChanged、CurrentSongChanged）是否生效
- **设置无法保存**
  - 检查配置文件权限、磁盘空间；确认 ConfigurationService 的保存操作未抛出异常
- **专辑封面不显示**
  - 检查 AlbumArtPath 路径有效性；确认 HasAlbumArt 属性正确更新
- **收藏功能异常**
  - 确认 IsFavorite 属性的响应式更新；检查收藏状态的持久化

## 结论
本数据模型以简洁、清晰为核心设计原则：AppSettings 提供应用程序设置管理；Song 聚焦增强的元数据与文件定位；Playlist 提供播放队列与导航策略；PlayState 与 PlaybackMode 描述播放状态与行为；PlayerStatus 封装播放器关键状态并采用不可变设计。新增的响应式属性支持使 UI 交互更加流畅，配置持久化服务确保用户偏好的一致性。服务层与视图模型围绕这些模型协同工作，形成完整的播放体验闭环。建议在后续迭代中进一步优化大库场景下的性能与内存占用、完善错误提示与日志记录、增强设置的版本兼容性处理。

### 实际使用示例与最佳实践
- **创建并使用播放列表**
  - 通过 IPlaylistService.CreatePlaylist 与 SetCurrentPlaylist 初始化播放列表
  - 使用 AddSongToPlaylist/RemoveSongFromPlaylist 管理歌曲集合
- **切换播放模式**
  - 通过绑定 Repeat/ Shuffle 按钮切换 PlaybackMode，实现 Loop 与 Shuffle
  - Normal 作为默认态，保持用户预期的一致性
- **播放控制**
  - 通过 IMusicPlayerService 的 Play/Pause/Resume/Stop/Seek 控制播放
  - 订阅 PlaybackStateChanged 与 PositionChanged 更新 UI
- **元数据扫描**
  - 使用 FileScannerService 扫描指定目录，支持进度与取消；异常文件降级为最小可用 Song
- **库管理**
  - 使用 IMusicLibraryService 的 Songs/FilteredSongs 进行展示与过滤
- **设置管理**
  - 通过 IConfigurationService 管理扫描文件夹、主题、音量等设置
  - SettingsViewModel 中的响应式属性确保设置界面与配置的实时同步
- **收藏功能**
  - 使用 Song.IsFavorite 属性管理收藏状态
  - 收藏状态应与用户偏好设置一起持久化
