# LocalMusicPlayer PRD

## 1. 项目概述

- **项目名称**：LocalMusicPlayer
- **类型**：桌面应用（本地音乐播放器）
- **核心功能**：扫描本地文件夹、管理播放列表、支持多种音频格式的跨平台音乐播放
- **目标用户**：需要播放本地音乐文件的用户

## 2. 技术栈

| 组件 | 技术 |
|------|------|
| UI 框架 | Avalonia 11.3.8 |
| MVVM | ReactiveUI 11.3.8 |
| 音频播放 | LibVLCSharp 3.8.1 |
| 元数据读取 | TagLibSharp 2.3.0 |
| 目标平台 | Windows / macOS / Linux |

## 3. 功能需求

### 3.1 音乐扫描 ✅

- 选择本地文件夹，递归扫描支持的音频文件
- 自动读取文件元数据（标题、艺术家、专辑、时长）
- 支持格式：MP3, FLAC, WAV, AAC, OGG, M4A

### 3.2 播放控制 ✅

- 播放 / 暂停 / 停止 / 上一曲 / 下一曲
- 进度条显示和拖拽跳转
- 音量调节（静音 / 增减）
- 双击歌曲立即播放

### 3.3 播放列表 ✅

- 显示歌曲列表（标题 / 艺术家 / 时长）
- 当前播放高亮
- 支持键盘上下选择

### 3.4 搜索 ✅

- 实时搜索歌曲（按标题、艺术家模糊匹配）
- 列表动态过滤

### 3.5 播放队列 ✅

- 临时播放队列
- 队列管理（移除、清空）
- 当前播放歌曲同步

### 3.6 播放模式 ✅

- **正常播放**：按顺序播放
- **随机播放**：随机打乱顺序
- **列表循环**：循环播放整个列表

### 3.7 歌词显示 ✅

- 自动加载同目录 LRC 格式歌词文件
- 歌词同步滚动显示
- 无歌词时显示专辑封面

### 3.8 专辑封面 ✅

- 从音频文件元数据提取专辑封面
- 默认封面占位图

### 3.9 收藏功能 ✅

- 收藏/取消收藏歌曲
- 收藏状态持久化

## 5. 非功能需求

### 5.1 性能要求

- 启动时间 < 3 秒
- 扫描1000首歌曲 < 10 秒
- 播放过程无卡顿
- 内存占用 < 200MB

### 5.2 兼容性

- Windows 10/11、macOS 10.15+、Ubuntu 20.04+
- 支持Unicode文件名和元数据

### 5.3 可靠性

- 异常文件自动跳过
- 播放错误自动尝试下一首
- 配置自动备份

## 7. 技术实现

### 7.1 架构模式

- **MVVM**：ViewModel 继承 ReactiveObject
- **依赖注入**：服务在 App.axaml.cs 注册
- **响应式编程**：使用 ReactiveUI 的 ObservableAsPropertyHelper 和 ReactiveCommand

### 7.2 核心服务

| 服务 | 职责 |
|------|------|
| IMusicPlayerService | 音频播放控制（LibVLC 封装） |
| IPlaylistService | 播放列表管理与播放模式 |
| IMusicLibraryService | 音乐库数据管理 |
| IScanService | 文件夹扫描与元数据读取 |
| ILyricsService | 歌词加载与解析 |
| IAlbumArtService | 专辑封面提取与缓存 |
| IConfigurationService | 应用配置持久化 |

### 7.3 数据模型

- **Song**：歌曲元数据（标题、艺术家、专辑、时长、路径等）
- **Playlist**：播放列表（歌曲集合、当前索引）
- **PlaybackMode**：播放模式枚举（Normal/Shuffle/Loop）

## 8. 已知限制

- LibVLC 依赖，应用体积较大（~50MB）
- DRM 保护文件无法播放
- Linux 平台媒体键支持可能不完整
- 纯本地播放器，不支持流媒体
