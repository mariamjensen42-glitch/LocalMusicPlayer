# Git 提交规范

## 提交信息格式

```
<类型>: <简短描述>

[可选的详细说明]
```

## 类型标识

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复 bug |
| `refactor` | 重构（不改变功能） |
| `style` | 代码格式调整 |
| `docs` | 文档更新 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具变更 |

## 规则

1. 标题不超过 50 字符
2. 使用中文描述，简明扼要
3. 标题句尾不加句号
4. 详细说明与标题之间空一行
5. 详细说明每行不超过 72 字符

## 示例

```
feat: 添加歌曲搜索功能

支持按歌名、歌手、专辑搜索
```
