# 歌词解析和渲染实现计划

## 目标
参考 original-sound-hq-player-win2d-navi 项目，为 LocalMusicPlayer（基于 Avalonia）实现歌词逐字高亮动画和自动滚动功能。

## 当前状态分析

### 已有功能
- **LyricsService** - 歌词解析（已实现 LRC 解析、时间戳匹配、翻译匹配）
- **LyricLine** - 歌词数据模型（Timestamp, Text, Translation, LineAnimateDuration, IsActive）
- **PlayerPageViewModel** - 歌词管理（更新当前索引、触发滚动）
- **BoolToLyricColorConverter** - 当前行颜色切换

### 缺失功能
1. **逐字高亮动画** - 使用 Composition API 实现歌词从右到左的裁剪动画
2. **自动滚动 Behavior** - 歌词切换时自动滚动到当前行并居中

## 实现步骤

### 1. 创建 LyricLineControl 歌词行控件
**文件**: `Controls/LyricLineControl.cs`

参考 original-sound-hq-player-win2d-navi 的 LyricsLineControl：
- 创建 `LyricLineControl : UserControl`
- 添加依赖属性：
  - `LyricText` (string)
  - `TranslateText` (string)
  - `TranslateVisibility` (Visibility)
  - `IsCurrentLine` (bool) - 触发高亮动画
  - `LineAnimateDuration` (TimeSpan) - 动画持续时间
  - `LyricsFontSize` (double)
  - `TranslateFontSize` (double)
- 实现 `StartTimerAnimation()` - 使用 Avalonia Composition 实现逐字高亮
  - 创建 `InsetClip` 或使用 `Clipping` 模拟
  - 创建 `ScalarKeyFrameAnimation` 从右到左裁剪
  - 动画时长 = `LineAnimateDuration`

### 2. 创建 LyricsAutoScrollBehavior 自动滚动行为
**文件**: `Behaviors/LyricsAutoScrollBehavior.cs`

参考 original-sound-hq-player-win2d-navi 的 LyricsAutoScrollBehavior：
- 创建 `LyricsAutoScrollBehavior : AvaloniaObject`
- 订阅 `IsCurrentLineChanged` 事件
- 当当前行变化时，计算目标偏移量并平滑滚动

### 3. 创建歌词相关 Converter
**文件**: `Converters/BoolToLyricFontWeightConverter.cs`
- 当前行使用 Bold 字体，非当前行使用 Normal

**文件**: `Converters/TranslationVisibilityConverter.cs`

### 4. 创建/更新 PlayerPageView
**文件**: `Views/Player/PlayerPageView.axaml`

参考 PlayerView 的布局，添加歌词显示区域：
- 使用 `ItemsControl` 或 `ListBox` 显示歌词

### 5. 创建 PlayerPageView 代码后台
**文件**: `Views/Player/PlayerPageView.axaml.cs`

## 技术要点

### 逐字高亮动画原理
```
歌词文本 → 裁剪层(Clipping) → 动画: RightInset 从 100% 渐变到 0%
```
在 Avalonia 中可以使用：
- `CustomDrawingOperation` 实现裁剪
- 或使用 `Border.Clip` 属性
- 或使用 `Transform` + `Clip` 组合

### 自动滚动原理
```csharp
当 IsCurrentLine = true 时：
1. 获取当前歌词行在列表中的位置
2. 计算中心偏移量 = 容器高度/2 - 行高度/2
3. 使用 ScrollViewer.ScrollTo() 平滑滚动
```
